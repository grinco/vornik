# Reference

Authoritative lookup material for running vornik. Use these pages when you
already understand the concepts and need exact key names, defaults, and
command syntax.

- **[Configuration](configuration.md)** — the customer-relevant `config.yaml`
  keys and feature flags, organised by section, with types and defaults.
- **[vornikctl CLI](vornikctl.md)** — the `vornikctl` commands you use day to day
  to inspect and control a running daemon: tasks, projects, keys, memory,
  diagnostics, and more.

If you are getting started for the first time, begin with the
[Getting Started guide](../getting-started/index.md) and the
[concepts overview](../concepts/index.md), then return here for specifics.
For end-to-end walkthroughs, see the [guides](../guides/index.md). If
something is misbehaving, the [troubleshooting](../troubleshooting/index.md)
page is the place to start.
