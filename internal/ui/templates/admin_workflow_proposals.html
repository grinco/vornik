Workflow Proposals

← Admin

Architect-emitted proposals to evolve workflow YAML. Review the proposed change, then approve or reject. Approving records the decision but does NOT write the file — click Apply on the drill-down page (or POST to /api/v1/admin/workflow-proposals/{id}/apply) to write the new YAML to both config trees, git-commit it in the source tree, and broadcast a config-reload to every replica. Applied changes are reversible via the Rollback button.
